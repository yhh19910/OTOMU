💻 技術實現對比

CSS 版本（簡單）

.photo-grid { column-count: 3; column-gap: 1.5rem; column-fill: balance; } .photo-item { break-inside: avoid; margin-bottom: 1.5rem; }

JavaScript 版本（精確）

const initMasonry = () => { const container = galleryRef.value; const items = container.querySelectorAll('.photo-item'); const gap = 24; const containerWidth = container.offsetWidth; // 動態計算最佳列數 const columns = Math.max(1, Math.floor((containerWidth + gap) / (300 + gap))); const columnWidth = (containerWidth - (columns - 1) * gap) / columns; // 追蹤每列的高度 const columnHeights = new Array(columns).fill(0); items.forEach((item) => { // 找到最短的列 const shortestColumnIndex = columnHeights.indexOf(Math.min(...columnHeights)); // 精確計算位置 const x = shortestColumnIndex * (columnWidth + gap); const y = columnHeights[shortestColumnIndex]; // 設置絕對位置 item.style.position = 'absolute'; item.style.left = `${x}px`; item.style.top = `${y}px`; item.style.width = `${columnWidth}px`; // 更新列高度 columnHeights[shortestColumnIndex] += item.offsetHeight + gap; }); };

🤔 何時選擇JavaScript版本？

✅ 適合JavaScript版本的情況：

  • 需要完美的列高度平衡
  • 想要複雜的動畫效果
  • 需要精確控制照片順序
  • 要整合虛擬滾動
  • 需要動態添加/移除照片
  • 要支持拖拽重排

⚠️ 保持CSS版本的情況：

  • 追求最佳性能
  • 照片數量不多（<50張）
  • 不需要複雜互動
  • 希望代碼簡潔
  • SEO優先考量
  • 支援舊瀏覽器

🎯 建議

對於您的攝影作品集網站，我建議：

🏆 推薦方案：保持CSS版本

因為您的網站：

  • 主要展示攝影作品，重點是照片本身
  • CSS版本已經能很好地保持照片比例
  • 性能優秀，載入速度快
  • 代碼簡潔，易於維護

除非您需要以下功能，否則不建議切換到JavaScript版本：

  • 照片點擊放大的燈箱效果
  • 無限滾動載入更多照片
  • 照片分類篩選動畫
  • 拖拽重新排序功能
